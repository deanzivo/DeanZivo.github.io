我的零基础hexo之路(开篇)

Posted on 2019-03-31 | Edited on 2019-04-07 | In 网站技术

我不是程序员，却想着干点程序员干的事情。在找一些技术文章的时候，偶然发现的”hexo”这个玩意，原来很多程序员都用这个玩意儿写博客。我也被hexo主题里面的 next 风格页面所吸引。

恩，有点心动，好吧，我也来搭建一个吧。

一路苦逼的啥也不懂，到处找资料查数据，一点一点的慢慢瞎打瞎晃，终于到今天正式把这个博客弄起来，也费了不少力气。 这里要交代一下的是，我只是把我的MacBook Pro搞定了，公司里面一台PC的win7，我实在搞不过来，简而言之，还是macOS系统稍微好弄一点。

网上hexo技术很多，我这里也不赘述，我之所以写，是怕以后我重新弄的时候，又被弄的糊里糊涂的，算是写一个安装记录吧，以后也可以自己查询回忆一下。

  1. 首先是安装Git、Node.js和hexo、还有Typora，我是直接去官网下了安装包git-2.21.0、node-v10.15.3和Typora，Typora是编辑软件，就是我现在正在打字用的软件。
    地址如下：
    Gti https://sourceforge.net/projects/git-osx-installer/
    Node.js https://nodejs.org/en/
    装好以后 执行：node -v 和 git —version出来版本号就表示好了。

  2. 开始安装 hexo ，( 可以参考官网文档: https://hexo.io/zh-cn/ )

    • npm install hexo-cli -g
    • hexo init blog
    • cd blog
    • npm install
    • hexo server

    我用的命令如下 ：
    npm install -g hexo-cli 或者npm install hexo-cli -g
    这里提醒一下，最好用
    sudo npm install hexo-cli -g 我试过几次，sudo可以避免权限不足 。
    mkdir blog 创建文件夹
    cd blog 进入目录（ 这里blog可以换成你想用的文件夹名字 ）
    hexo init 在blog文件夹里安装hexo 如果报错加 sudo

  3. 如果以上都装好了，通过本地服务就可以看hexo装的行不行。
    hexo s 或 hexo server 开启本地服务

    运行成功会跳出：http://localhost:4000/ 可以本地浏览网站了。

    我操作的时候服务报错，错误如下，原因其实是后面关于修改文件没有注意空格的原因，基本上只要安装好，不修改任何东西，不会有什么问题。

    使用hexo s或hexo g终端命令是报错ERROR Process failed: _posts/name.md
    这个报错已经给了提示，在name.md文件的第一行有错误，基本是md语法错误。比如上图的报错是因为在title:的后面写的文章title里有:，所以报错

  4. 博客关联到Github仓库
    首先：进入Github官网，注册账号。创建仓库的名字必须为username.github.io，用户名为123456，因此我创建的仓库就是123456.github.io，这是很关键的一点，很重要。输入名字后，直接点最下面绿色的按钮，创建新仓库。这一部分基本完成了，接下去需要在终端操作。

    (这里原来有几张图，Typora上图比较麻烦，等研究好了再说吧)

这波操作做完了，就下面一波操作了。

首先我们要编辑图中_config.yml文件，这是博客的主要配置文件，在下面一部分，我们要频繁使用这个配置文件，这里先编辑一步，为了管理GitHub账号。

进入你的hexo文件夹根目录下，用编辑器打开_config.yml，修改deploy项，如下（根据你的github进行相应更改）

这个就是在_config.yml文件的最下面，是没有完成的部分，直接找到直接复制就行。

  1. 执行产生静态页面命令
    hexo g 或者 hexo generate 这一步基本上不会有什么错误

  2. 然后进行部署到服务器
    hexo d 或者hexo deploy 这一步或许会报错，执行以下命令

    npm install–save hexo-deployer-git 或 npm install hexo-deployer-git —save

    然后你需要输入你GitHub的用户名和密码，这样你就就可以使用你的仓库名去访问你的博客主页了。为了每一次部署不必一种输入密码，我们可以生成秘钥，然后提交到GitHub，进行关联，那么你下次就不需要再输入密码了。

    这里缺了一个域名注册及关联的事情，我不需要域名，因此就没有做自有域名这一块的事情。如果需要再自己搜索吧。

    关联GitHub账号，免密提交。

    点击箭头的按钮，进入设置，然后点击SSH and GPG keys，创建新的SSH，将之前复制的公钥黏贴，提交，OK。

    (这里原来有几张图，Typora上图比较麻烦，等研究好了再说吧)

    注意：私有秘钥和公众秘钥，不要选错了，复制去网上的公有秘钥。

  3. 基本上这样就完事了，可以创建文章等操作了。命令如下：

    hexo new "postName"#新建文章
    hexo new page "pageName"#新建页面
    hexo generate#生成静态页面至public目录
    hexo server#开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
    hexo deploy#将.deploy目录部署到GitHub
    hexo help# 查看帮助
    hexo version#查看Hexo的版本

  4. 后面就是关于选hexo的主题和用Typora编辑md文章的事情了。

    我只是简单的用了 next 的主题，并开始了这篇文章的写作。

    等我详细了解并会调整 next主题和 Typora后，再接着写第二篇。

  5. 最后把一些报错的事情整理一下。

    1.出现 error deployer not found:git 或者 error deployer not found:github 的错误
    解决办法：执行 npm install hexo-deployer-git --save

    2.出现类似 FATAL can not read a block mapping entry; a multiline key may not be an implicit key at line 70, column 1: … 错误
    解决办法：_config.yml中配置项的冒号后面要用空格隔开，再跟内容

    3.中文乱码问题
    解决办法：将文件的内容编码改为UTF8格式

    4.’hexo sever’能够成功运行，但是localhost:4000无法访问
    解决办法：执行hexo s -p 5000，改用其他端口启动

  6. 最后我即将发布这篇文章，用了一些Typora的图片插入和文本格式。
